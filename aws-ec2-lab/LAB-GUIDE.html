1Amazon Linux 2 + Apache + 2 Apps

Launch an EC2 instance with Amazon Linux 2, install Apache (httpd), and host two web applications: FashionHub (Myntra-like) and ConnectPro (LinkedIn-like).

Internet
→
EC2 (Amazon Linux 2)
→
Apache :80
→
fashionhub.html
connectpro.html

Step 1: Launch EC2 Instance

AWS CONSOLE

Go to EC2 → Launch Instance and configure:


Name: task1-apache-server
AMI: Amazon Linux 2023 (Free tier)
Instance type: t2.micro
Key pair: Create new → task1-key
Security Group: Allow SSH (22), HTTP (80)
Storage: 8 GB gp2 (default)

Step 2: Connect to Instance

BASH 
# From your local machine (terminal / MobaXterm / PuTTY)
# First make sure the .pem key has correct permissions
chmod 400 task1-key.pem

# Connect via SSH (replace with your EC2 Public IP)
ssh -i task1-key.pem ec2-user@<YOUR_PUBLIC_IP>

Step 3: Install Apache (httpd)

BASH 
# Update the system packages
sudo yum update -y

# Install Apache web server
sudo yum install -y httpd

# Start Apache service
sudo systemctl start httpd

# Enable Apache to start on boot
sudo systemctl enable httpd

# Check Apache is running (should show active/running)
sudo systemctl status httpd
✅ Apache web root is at: /var/www/html/ — this is where you upload your HTML files

Step 4: Upload Web Applications

🛍
FashionHub
fashionhub.html
Myntra-inspired fashion store
💼
ConnectPro
connectpro.html
LinkedIn-inspired professional network
BASH — Option A: Upload from your local machine using SCP 
# Upload both HTML files from your local machine
scp -i task1-key.pem fashionhub.html ec2-user@<IP>:/home/ec2-user/
scp -i task1-key.pem connectpro.html ec2-user@<IP>:/home/ec2-user/

# Then SSH in and move files to web root
sudo mv /home/ec2-user/fashionhub.html /var/www/html/
sudo mv /home/ec2-user/connectpro.html /var/www/html/
BASH — Option B: Create files directly on server using nano 
# Create fashionhub.html and paste HTML content
sudo nano /var/www/html/fashionhub.html
# (Ctrl+X → Y → Enter to save)

# Create connectpro.html and paste HTML content
sudo nano /var/www/html/connectpro.html

Step 5: Set File Permissions

BASH 
# Give Apache ownership of web root files
sudo chown -R apache:apache /var/www/html/
sudo chmod -R 755 /var/www/html/

# Restart Apache to apply any changes
sudo systemctl restart httpd

# Verify files are present
ls -la /var/www/html/

Step 6: Access Your Applications

🌐 Open your browser and go to:

  http://<YOUR_PUBLIC_IP>/fashionhub.html   → FashionHub app
  http://<YOUR_PUBLIC_IP>/connectpro.html   → ConnectPro app
⚠️ Make sure port 80 (HTTP) is open in the Security Group Inbound Rules. If not, go to EC2 → Security Groups → Edit Inbound Rules → Add HTTP

2Ubuntu + Nginx + 2 Apps

Launch a second EC2 instance with Ubuntu 22.04, install Nginx, and host two applications: SoundWave (Spotify-like) and StreamMax (Netflix-like).

Internet
→
EC2 (Ubuntu 22.04)
→
Nginx :80
→
soundwave.html
streammax.html

Step 1: Launch EC2 Instance

AWS CONSOLE
Name: task2-nginx-ubuntu
AMI: Ubuntu Server 22.04 LTS (Free tier)
Instance type: t2.micro
Key pair: Create new → task2-key
Security Group: Allow SSH (22), HTTP (80)
Username to SSH: ubuntu (not ec2-user!)

Step 2: Connect to Ubuntu Instance

BASH 
chmod 400 task2-key.pem

# Note: Ubuntu uses 'ubuntu' as username, not 'ec2-user'
ssh -i task2-key.pem ubuntu@<YOUR_PUBLIC_IP>

Step 3: Install Nginx

BASH 
# Update Ubuntu package list
sudo apt update

# Install Nginx web server
sudo apt install -y nginx

# Start Nginx
sudo systemctl start nginx

# Enable Nginx on boot
sudo systemctl enable nginx

# Check status (should be active/running)
sudo systemctl status nginx
✅ Nginx web root on Ubuntu is at: /var/www/html/ (same as Apache)

Step 4: Upload Web Applications

🎵
SoundWave
soundwave.html
Spotify-inspired music streaming
🎬
StreamMax
streammax.html
Netflix-inspired video streaming
BASH — Upload using SCP 
# From local machine — upload files (use ubuntu@ for Ubuntu instances)
scp -i task2-key.pem soundwave.html ubuntu@<IP>:/home/ubuntu/
scp -i task2-key.pem streammax.html ubuntu@<IP>:/home/ubuntu/

# SSH in and move to Nginx web root
sudo mv /home/ubuntu/soundwave.html /var/www/html/
sudo mv /home/ubuntu/streammax.html /var/www/html/

# OR create files directly with nano
sudo nano /var/www/html/soundwave.html
sudo nano /var/www/html/streammax.html

Step 5: Set Permissions and Restart

BASH 
# Set ownership to www-data (Nginx user on Ubuntu)
sudo chown -R www-data:www-data /var/www/html/
sudo chmod -R 755 /var/www/html/

# Restart Nginx
sudo systemctl restart nginx

# Test Nginx config (no errors means it's good)
sudo nginx -t

Step 6: Access Your Applications

🌐 Open your browser and go to:

  http://<YOUR_PUBLIC_IP>/soundwave.html   → SoundWave app
  http://<YOUR_PUBLIC_IP>/streammax.html   → StreamMax app

3Amazon Linux + Nginx + Portfolio

Launch a third EC2 instance with Amazon Linux, install Nginx, and deploy a developer Portfolio website.

Internet
→
EC2 (Amazon Linux)
→
Nginx :80
→
index.html
(Portfolio)

Step 1: Launch EC2 Instance

AWS CONSOLE
Name: task3-portfolio
AMI: Amazon Linux 2023 (Free tier)
Instance type: t2.micro
Security Group: Allow SSH (22), HTTP (80), HTTPS (443)

Step 2: Install Nginx on Amazon Linux

BASH 
# Connect as ec2-user
ssh -i task3-key.pem ec2-user@<YOUR_PUBLIC_IP>

# Update packages
sudo yum update -y

# Install Nginx (Amazon Linux uses amazon-linux-extras or yum)
sudo yum install -y nginx

# Start and enable Nginx
sudo systemctl start nginx
sudo systemctl enable nginx
sudo systemctl status nginx

Step 3: Deploy Portfolio as index.html

🧑‍💻
Developer Portfolio
portfolio.html → rename to index.html
Cloud & DevOps Engineer personal portfolio website
BASH 
# Upload portfolio.html from local machine
scp -i task3-key.pem portfolio.html ec2-user@<IP>:/home/ec2-user/

# Move and rename as index.html (so it opens at root URL)
sudo mv /home/ec2-user/portfolio.html /var/www/html/index.html

# Remove default Nginx page (optional)
sudo rm -f /usr/share/nginx/html/index.html

# Set permissions
sudo chown nginx:nginx /var/www/html/index.html
sudo chmod 644 /var/www/html/index.html

# Restart Nginx
sudo systemctl restart nginx

Step 4: Access Portfolio

🌐 Open your browser:

  http://<YOUR_PUBLIC_IP>   → Portfolio homepage (index.html loads automatically)

  • Security Group must allow HTTP port 80 for websites to be accessible
